# Видеочат-рулетка (Video Chat Roulette)

Приложение для видеочата на базе Qt 6 с использованием WebRTC (библиотека libdatachannel) и сигнального сервера на Node.js.

## Возможности

- **P2P видеочат**: Прямое WebRTC-соединение между пользователями.
- **Случайный подбор (Roulette)**: Поиск случайного собеседника через сигнальный сервер.
- **Современный интерфейс**: Темная тема, реализованная на Qt Quick (QML).
- **Кроссплатформенность**: Поддержка Windows, macOS и Linux.

## Архитектура

### Клиент (Qt 6)
- **CameraHandler**: Управляет захватом видео, используя `QMediaCaptureSession`.
- **SignalingClient**: WebSocket-клиент для связи с сигнальным сервером.
- **WebRTCManager**: Обертка для работы с WebRTC на базе `libdatachannel`.
- **VideoChatApp**: Основная логика приложения и управление состояниями.

### Сигнальный сервер (Node.js)
- Сервер на базе WebSockets.
- Алгоритм случайного подбора партнеров.
- Ретрансляция SDP-офферов и ICE-кандидатов.
- Управление активными соединениями.

## Требования

### Клиент
- Qt 6.6+ со следующими модулями:
  - Core
  - Widgets
  - Multimedia
  - Quick
  - WebSockets
- CMake 3.20+
- Компилятор с поддержкой стандарта C++17
- libdatachannel (автоматически загружается через CMake)

### Сервер
- Node.js 14+
- npm

## Инструкция по сборке

### 1. Сборка клиента

```bash
# Перейдите в директорию проекта
cd VideoChatRoulette

# Создайте директорию для сборки
mkdir build
cd build

# Конфигурация через CMake
cmake ..

# Сборка
make -j$(nproc)  # Для Linux/macOS
# или
cmake --build .  # Кроссплатформенный вариант
```

### 2. Настройка и запуск сигнального сервера

```bash
# Перейдите в директорию сервера
cd server

# Установите зависимости
npm install

# Запустите сервер
npm start
```

По умолчанию сервер запустится на `ws://localhost:8080`.

## Использование

1. **Сначала запустите сигнальный сервер**:
   ```bash
   cd server && npm start
   ```

2. **Запустите клиентское приложение**:
   ```bash
   ./build/VideoChatRoulette
   ```

3. **Работа с программой**:
   - Нажмите **"Connect"**, чтобы подключиться к сигнальному серверу.
   - Нажмите **"Start Chat"** для поиска собеседника.
   - После установки соединения вы увидите видео партнера.
   - Нажмите **"Next"**, чтобы найти нового человека.
   - Нажмите **"Stop"**, чтобы завершить текущий чат.

## Конфигурация

### URL сервера
По умолчанию клиент подключается к `ws://localhost:8080`. Вы можете изменить этот адрес в интерфейсе или в исходном коде.

### STUN-сервер
Приложение использует STUN-сервер Google (`stun:stun.l.google.com:19302`) для обхода NAT. Это можно изменить в файле `WebRTCManager.cpp`.

## Сигнальный протокол

### Сообщения клиента

#### Поиск партнера
```json
{
  "type": "search",
  "id": "client-id"
}
```

#### WebRTC Offer (Предложение)
```json
{
  "type": "offer",
  "to": "partner-id",
  "from": "my-id",
  "sdp": "offer-sdp"
}
```

#### WebRTC Answer (Ответ)
```json
{
  "type": "answer",
  "to": "partner-id",
  "from": "my-id",
  "sdp": "answer-sdp"
}
```

#### ICE Candidate
```json
{
  "type": "ice-candidate",
  "to": "partner-id",
  "from": "my-id",
  "candidate": "candidate-string",
  "sdpMid": "0",
  "sdpMLineIndex": 0
}
```

### Сообщения сервера

#### Назначение ID клиента
```json
{
  "type": "id",
  "id": "generated-client-id"
}
```

#### Партнер найден
```json
{
  "type": "partner-found",
  "partnerId": "partner-client-id"
}
```

#### Партнер отключился
```json
{
  "type": "partner-disconnected"
}
```

## Устранение неполадок

### Не работает камера
- Убедитесь, что приложению выданы разрешения на использование камеры.
- Проверьте, не занята ли камера другим приложением.
- Убедитесь, что бэкенд Qt Multimedia корректно установлен в системе.

### Проблемы с соединением
- Проверьте, запущен ли сигнальный сервер.
- Проверьте настройки брандмауэра (файрвола) для WebSocket-соединений.
- Убедитесь, что есть доступ к STUN-серверу.

### Проблемы при сборке
- Убедитесь, что установлена версия Qt 6.6+ со всеми необходимыми модулями.
- Проверьте версию CMake (минимум 3.20).
- Убедитесь, что ваш компилятор поддерживает C++17.
